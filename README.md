## Мотивация и цель

Методы обучения с подкреплением позволяют формировать адаптивные торговые стратегии на основе взаимодействия агента с рыночной средой — без явного задания правил. Однако сравнение классических табличных алгоритмов в единых, воспроизводимых условиях остаётся актуальной задачей.

**Цель работы** — честно и воспроизводимо сравнить четыре табличных RL-алгоритма (Q-Learning, SARSA, SARSA(λ), Monte Carlo) в рамках единой торговой среды с реалистичными ограничениями, оценив их по финансовым метрикам и устойчивости к изменению рыночных условий.

## Среда и постановка задачи

Задача формализована как марковский процесс принятия решений (MDP). Реализован кастомный Gymnasium-совместимый класс `MyTradingEnv`.

**Действия (3):**

| Код | Действие |
|-----|----------|
| `0` | Hold — удержание позиции |
| `1` | Buy — открытие позиции |
| `2` | Sell — закрытие позиции |

**Пространство состояний (дискретное, 162 состояния):**

| Признак | Уровней | Описание |
|---------|---------|----------|
| RSI | 3 | Перепроданность / нейтраль / перекупленность |
| MACD diff | 3 | Медвежий / нейтральный / бычий сигнал |
| MA-тренд | 3 | Нисходящий / боковой / восходящий |
| Флаг позиции | 2 | Нет позиции / есть позиция |
| PnL-состояние | 3 | Убыток / около нуля / прибыль |

Итого: 3 × 3 × 3 × 2 × 3 = **162 уникальных состояния**.

**Функция награды** учитывает результат сделки (реализованный PnL) и конечный баланс, с усиленными штрафами за убыточные операции.

**Торговые ограничения среды:**

- Комиссия биржи и проскальзывание: **0.01%** каждое
- Максимальное время удержания позиции: **72 шага** (72 часа)
- Максимальная допустимая просадка по позиции: **8%**
- Начальный капитал: **1 000 USD**

## Алгоритмы

Все агенты реализованы на основе единого абстрактного класса `BaseClassicalAgent` с общим интерфейсом (Q-таблица, ε-жадная стратегия с экспоненциальным затуханием, логирование).

### Классические RL-алгоритмы

**Q-Learning** — off-policy TD-метод. Обновляет функцию ценности на основе максимального ожидаемого значения в следующем состоянии, что обеспечивает более агрессивное исследование пространства.

**SARSA** — on-policy TD-метод. Обновляет оценки с учётом фактически выбранного следующего действия, формируя более консервативную стратегию, осторожную в условиях неопределённости.

**SARSA(λ)** — расширение SARSA механизмом следов пригодности (eligibility traces, λ = 0.7). Ускоряет распространение информации о вознаграждении на ранее посещённые состояния, что повышает скорость начальной адаптации.

**Monte Carlo** — эпизодический метод. Обновляет функцию ценности только по завершении эпизода на основе фактического дисконтированного возврата, что обеспечивает несмещённые оценки, но при высокой дисперсии обучения.

### Базовые стратегии (Baseline)

- **Random** — случайный выбор действий
- **Buy & Hold** — купить и держать
- **Moving Average** — торговля по сигналам скользящих средних

## Данные и протокол обучения

**Данные:** исторические часовые свечи `BTC/USDT` (таймфреймы `1h`, `15m`, `1m`).

- **Обучающая выборка:** котировки за 2021 год
- **Тестовая выборка:** отдельный временной период, не использовавшийся при обучении

**Процедура обучения** состоит из двух последовательных этапов:

1. **Первичное обучение (Exploration phase):** 8 000 эпизодов, ε-жадная стратегия с ε от 1.0 до 0.01 (экспоненциальное затухание). Акцент на активное исследование пространства состояний.

2. **Дообучение (Fine-tuning phase):** 10 000 эпизодов на базе сохранённой модели, сниженная скорость обучения, повышенный коэффициент дисконтирования. Цель — стабилизация политики и улучшение качества торговых решений.

Периодическая валидация в процессе обучения проводилась в детерминированном режиме (без исследования).

## Метрики

| Метрика | Описание |
|---------|----------|
| Доходность, % | Итоговый прирост портфеля |
| Конечный баланс | Стоимость портфеля по завершении эпизода |
| Win Rate, % | Доля прибыльных сделок |
| Profit Factor | Отношение суммарной прибыли к суммарным убыткам |
| Sharpe Ratio | Отношение доходности к стандартному отклонению |
| Max DD сделок, % | Максимальная просадка по отдельной сделке |
| Max DD портфеля, % | Максимальная просадка портфеля за период |
| Кол-во сделок | Торговая активность агента |
| Среднее время удержания | Средняя длительность открытой позиции |

### Сравнительный анализ

**Q-Learning и SARSA($\lambda$)** продемонстрировали наилучшее соотношение доходности, риска и обобщающей способности. На тестовой выборке оба алгоритма показали доходность ~70% при Sharpe Ratio = 0.37 и практически идентичном торговом поведении. Этот результат объясняется сближением динамики обновлений SARSA(λ) при λ = 0.7 с off-policy оценкой, характерной для Q-Learning.

**SARSA** формирует чрезмерно консервативную стратегию: минимальное число сделок (3 на тесте), отрицательная или нулевая доходность. On-policy природа алгоритма усиливает осторожность поведения в условиях неопределённости.

**Monte Carlo** показал наименьшую устойчивость: высокая дисперсия при обучении, нестабильный перенос на тестовые данные. Базовая версия агента не совершила ни одной сделки на тестовой выборке.

**Дообучение** (Fine-tuning) оказало существенное положительное влияние на TD-алгоритмы — особенно на Q-Learning (доходность на тесте выросла с 47.1% до 69.7%) и SARSA(λ) (с 37.0% до 70.1%). Для SARSA и Monte Carlo эффект оказался ограниченным.

## Как воспроизвести

Минимальный пример запуска (см. `notebooks/test_train.ipynb`):

```python
import pandas as pd
from training import TrainingManager, TrainingConfig
from agents.classical.qlearning_agent import QLearningAgent

# Данные
df = pd.read_csv("data/data_1h_2021.csv")

# Агент и конфиг
agent = QLearningAgent()
config = TrainingConfig(agent_name="QLearning", agent_type="QLearning")

# Обучение
manager = TrainingManager()
manager.train_agent(agent, df, config, experiment_name="qlearning_1h_2021")
```

Результаты сохраняются в `training_data/<experiment_name>/` в виде CSV-логов и чекпойнтов модели.

## Ограничения

- **Дискретизация признаков** неизбежно приводит к потере части информации о рыночной динамике и ограничивает выразительную способность модели.
- **Один инструмент и период:** обучение и тестирование проводились исключительно на BTC/USDT; переносимость на другие активы и таймфреймы не гарантирована.
- **Упрощённая модель исполнения:** не учитываются изменения ликвидности, влияние объёма на цену, задержки исполнения ордеров и иные микроструктурные эффекты.
- **Статичность стратегии:** после завершения обучения политика агента фиксируется и не адаптируется к структурным изменениям рынка в реальном времени.

Выявленные ограничения указывают на перспективные направления развития: переход к методам глубокого RL (DQN, Dueling DQN, Actor-Critic), расширение пространства признаков и обучение на нескольких финансовых инструментах.
